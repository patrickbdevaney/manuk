A browser engine you can see inside.

Manuk — written from scratch in Rust — compiled to WebAssembly and executing on this page. It is parsing, cascading, laying out and rasterizing real websites in your browser right now. Point at anything and it will show you its own computed style and layout box, straight out of Stylo and Taffy. Click a link and it will navigate.

Real: Stylo (Firefox's cascade), Taffy (flex/grid), tiny-skia (the rasterizer), and this engine's own DOM, inline/float/table layout, paint, hit-testing and stacking. Scrolling re-renders. The inspector reads the actual engine state — nothing is re-derived for display.
Not real: no JavaScript, and no live fetching — these are bundled snapshots of real sites, with their stylesheets inlined. Saying so is the only thing that makes the rest believable.

Why no JavaScript? (the honest, technical answer)

The engine embeds SpiderMonkey natively — real JS, real JIT. But this demo targets wasm32-unknown-unknown, and SpiderMonkey is C++: mozjs's FFI links a native library that has no build for that target.

SpiderMonkey can run on WebAssembly — but only as an interpreter, because WebAssembly forbids runtime code generation, which is precisely what a JIT is. That is why the Bytecode Alliance built a portable baseline interpreter for it. It also ships as a separate WASI module, not something linkable into this Rust wasm binary.

So: shipping JS here is a project, not a feature — and pretending otherwise would undermine everything else on this page. What you are looking at is the rendering engine, in full.

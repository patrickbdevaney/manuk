# Not a favourites list — a SAMPLE OF THE WEB. Twenty sites was an anecdote: every class bug we ever
# found was found because some site in the corpus happened to use that pattern, which means the bugs
# we had NOT found were exactly the ones no corpus site happened to use. That is a structural blind
# spot, not a coverage gap, and no amount of staring at twenty sites closes it.
#
# The CLASS column is what makes the cluster ranking a ledger instead of a list of grievances: a
# divergence that appears across `docs` and `saas` and `news` is a design-pattern bug, while one that
# appears on a single site is that site's custom CSS. Rank by distinct sites AND distinct classes.
#
# Sites are the ENTRY PAGE (the thing a person actually lands on), fetched once and fed to BOTH
# engines — the one-snapshot rule (oracle.rs), which is not optional and is enforced in code.
#
# Bot walls, logins and error pages are DISCARDED by `oracle_is_healthy`, never scored. A degraded
# oracle scores its own emptiness as our bug, and that has happened here before.

# SPA corpus — the Framework Exception Miner's input (METHODOLOGY Part 9, Part 21.2 item 3)

**This is the largest unmeasured unknown in the entire schedule, and it is binary.** If the substrate
React/Vue/Next need from us is additive IDL and scheduling work, the app web is fast to reach. If
hydration failure cascades into needing a scheduling-fidelity subsystem, it is not. You cannot plan
around that distinction while it is unmeasured — and the measurement is cheap, which is exactly why
leaving it unmeasured was indefensible.

The apps here are **real framework output**, not hand-written toys: a toy exercises the IDL *we thought
to implement*, which is a tautology, not a measurement. A real bundle exercises what the framework
actually calls on boot, which is the only thing that matters.

The miner does not check that these render correctly. It checks **what they throw**. Every exception is
a missing piece of substrate, named by the framework itself — the browser telling us its own bugs out
loud, which is a discovery mechanism no amount of spec-reading replaces.
